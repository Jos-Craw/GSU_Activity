# GSU Activity🎓📅
> Веб-приложение для автоматизации учета и записи на университетские мероприятия.

## 📌 О проекте
Приложение разработано для упрощения оповещения студентов о мероприятиях и автоматизации процесса регистрации. Проект вырос из курсовой работы (Physicalgram) и перерос в полноценную систему управления активностями факультета.

## 🚀 Технические особенности и алгоритмы
**Динамический учет мест:** Реализован сложный алгоритм обработки заявок (`zapisg`), который учитывает актуальное количество свободных мест в реальном времени при записи и отписке пользователей.
**Типы регистраций:** Поддержка как одиночных, так и групповых заявок.
**Архитектура БД:** Спроектирована реляционная структура данных (ER-диаграмма включает таблицы пользователей, ролей, новостей, типов постов и комментариев). 
**Безопасность:** Использование встроенных механизмов Django Auth и защиты «из коробки».

## 🛠 Технологический стек
**Backend:** Python, Django Framework (выбран за простоту и обилие библиотек). 
**Frontend:** HTML5, CSS3, Django-bootstrap (адаптивный дизайн для мобильных устройств). 
**Database:** SQLite3 (с возможностью миграции на PostgreSQL). 

## 👥 Ролевая модель
В системе реализовано 6 специализированных ролей администраторов для независимой модерации направлений: 
Культурный, Спортивный, Массовый, Трудовой, Психологический и Full Admin.
